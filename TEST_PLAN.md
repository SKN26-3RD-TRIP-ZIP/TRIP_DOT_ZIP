# [테스트 계획서] LangGraph 기반 지능형 여행 플래너 에이전트

## 1. 개요
* **목적**: 사용자의 여행 의도 파악부터 장소 검색, 일정 생성까지 이어지는 멀티 에이전트 워크플로우의 안정성 및 데이터 정합성 검증.
* **대상 시스템**: LangGraph 기반 LLM 에이전트 및 RAG(Self-Querying + Rerank) 검색 파이프라인.
* **핵심 지표**: Intent 라우팅 정확도, 목적지 변경 시 상태 초기화 여부, 검색 결과의 시맨틱 유사도.

### 6.1. 상세 결함 조치 리포트

#### 6.1.1 [Critical] 목적지 변경 시 데이터 오염(Data Pollution) 이슈
* **결함 내용**: '경주' 일정을 생성한 후 "부산으로 갈래"라고 변경했으나, 추천 리스트에 경주 장소가 섞여 나오는 현상.
* **원인 분석**: `select_places_node`에서 기존 데이터와 신규 목적지 간의 정합성 검증 단계가 미흡하여 이전 상태값을 그대로 반환함.
* **조치 내용**: 
    - `select_places_node` 내에 목적지 불일치 감지 로직 추가.
    - 목적지 변경이 감지되면 `itinerary`, `selected_places` 등을 즉시 초기화하도록 `Reducer` 로직 보완.
* **테스트 결과**: 목적지 전환 시 100% 확률로 데이터 초기화 및 신규 도시 데이터 로딩 확인 (**Pass**).

#### 6.1.2 [Major] 논리 연산자 우선순위에 따른 조건문 오작동
* **결함 내용**: 괄호 없는 논리식에서 `None` 값이 들어올 경우 의도치 않게 `True`로 평가되어 로직이 실행되거나 건너뜀.
* **원인 분석**: Python 내부 연산 우선순위로 인해 `(A and B) != C`가 아닌 `A and (B != C)` 형태로 해석됨.
* **조치 내용**: `is not None`을 활용한 명시적 비교문으로 수정하여 가독성과 논리적 정확도 확보.
* **테스트 결과**: 에지 케이스(None, 빈 값 입력 등) 시뮬레이션 결과 모두 정상 분기 확인 (**Pass**).

#### 6.1.3 [Major] 상세 지명 미인식으로 인한 목적지 누락
* **결함 내용**: "해운대 가고 싶어"라고 입력할 경우, '부산'이라는 키워드가 없어 목적지 추출에 실패함.
* **원인 분석**: 텍스트 매칭 기반의 단순 추출 로직 한계.
* **조치 내용**: `sub_locations` 매핑 사전을 도입하여 하위 지역명 입력 시 상위 행정구역으로 자동 치환하는 전처리 로직 구현.
* **테스트 결과**: 주요 관광지/동네 이름 입력 시 상위 도시가 정상적으로 State에 저장됨 (**Pass**).

#### 6.1.4 [Normal] 검색 결과의 시맨틱 매칭 품질 저하
* **결함 내용**: 단순 벡터 유사도 검색 결과가 사용자의 세부 요구사항(예: 조용한, 부모님과 함께)을 충분히 반영하지 못함.
* **원인 분석**: 임베딩 유사도만으로는 복합적인 뉘앙스 파악에 한계가 있음.
* **조치 내용**: 
    - `SelfQueryRetriever`를 통한 메타데이터 1차 필터링 수행.
    - `Reranker` 노드를 추가하여 상위 후보군을 사용자 쿼리 관점에서 재정렬.
* **테스트 결과**: 정성 평가 결과 추천 장소의 의도 부합도가 기존 대비 약 30% 향상됨 (**Pass**).

### 6.2. 종합 의견
초기 개발 단계에서 발생했던 **데이터 정합성** 및 **논리 연산 오류**를 모두 해결하였으며, 특히 멀티 에이전트 환경에서 `State` 관리를 최적화함으로써 사용자 인터랙션에 따른 유연한 대응이 가능해졌습니다. 이후 실사용자 피드백을 바탕으로 검색 쿼리를 지속적으로 고도화할 예정입니다.
